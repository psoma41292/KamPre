03 Code Quality Improvements

Bob enforced structural consistency and best practices across every file, removing entire categories of common quality defects before they entered the codebase.

3.1 Consistent Separation of Concerns

Bob enforced a strict three-layer frontend architecture: services (network I/O), components (UI only), and context (shared state). No component ever imports axios directly — all HTTP calls flow through api.js, making the network layer trivially mockable for tests. 

// api.js — all HTTP calls centralised here
const api = axios.create({ baseURL: "/api" });

// JWT interceptor — applied once, respected everywhere
api.interceptors.request.use((config) => {
  const token = localStorage.getItem("kampare_token");
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

3.2 Resilient Async Patterns

Bob chose Promise.allSettled() for concurrent platform price fetches, so a single upstream timeout never blocks the entire comparison response. This was not an afterthought — it was the pattern Bob produced on first generation. 

// priceService.js — platform failures isolated, never cascade
const settled = await Promise.allSettled(rawMock.map(enrichMock));
settled.forEach((o) => {
  if (o.status === "fulfilled") results.push(o.value);
});

3.3 Debounced Autocomplete Without Race Conditions

SearchBar.jsx uses a useRef-tracked debounce timer (200 ms) rather than a naive setTimeout, ensuring the timer is cleared on unmount and on every keystroke — preventing stale-closure suggestion leaks that are a common source of subtle UI bugs. 

// SearchBar.jsx — debounce via ref, cleaned up in useEffect
const debounceRef = useRef(null);
const loadSuggestions = useCallback((val) => {
  clearTimeout(debounceRef.current);
  debounceRef.current = setTimeout(async () => { … }, 200);
}, []);

useEffect(() => {
  loadSuggestions(query);
  return () => clearTimeout(debounceRef.current);  // ← cleanup
}, [query, loadSuggestions]);

3.4 Normalised Unit Comparison

Bob designed a pricePerUnit() helper that normalises all products to ₹ per 100 g/mL, enabling fair cross-platform comparison regardless of pack size. The groupByQuantity() function then tiers products so users compare like-for-like quantities — a data integrity decision that prevents misleading price rankings.

05 Security & IBM Policy Compliance

Bob applies IBM's mandatory security checklist to every code generation — not as a post-hoc review but as a real-time constraint during output.

5.1 Secrets Management

Every sensitive value — PORT, CACHE_TTL, SERPAPI_KEY, MONGODB_URI, JWT_SECRET — was placed in backend/.env (gitignored) with a safe .env.example template committed instead. Bob proactively generated the .gitignore to exclude .env.

5.2 Rate Limiting & Input Validation

express-rate-limit is applied to every /api/* route — 100 requests per 15-minute window — configured via environment variables so it is tunable per deployment tier without code changes.

5.3 JWT Authentication Architecture

Bob designed the AuthContext / LoginModal / api.js interceptor chain so that tokens are stored in localStorage, attached via an Axios request interceptor, and validated server-side in the auth.js middleware — following standard Bearer token flow rather than query-string secrets.

5.4 Information Disclosure Prevention

The global error handler in server.js returns only { error: "Internal server error" } to the client. Stack traces are logged server-side via console.error("[Error]", err.message) — never surfaced to the API consumer.
